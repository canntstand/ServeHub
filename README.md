# 🏗️ ServeHub. Personal Service Factory
> **Цель**: платформа для мгновенного развертывания персональных веб-сервисов с готовой DevOps-инфраструктурой, работающая на собственном железе

## Причины появления этого проекта
Проект является переосмыслением my-server. Предыдущий проект не был завершён, хотя и не заброшен окончательно. Его завершение возможно, но не раньше 2026 года. К такому исходу привели две основные причины:

1. **Потеря фокуса на основной цели**. Изначально меня интересовало создание инфраструктуры, куда я мог бы легко интегрировать что угодно с минимальными доработками. Однако со временем я неосознанно отошёл от этой идеи и понял, что трачу силы не на то, что было первостепенным. Вместо развития архитектуры и настройки систем я занимался рутинной и не самой интересной для меня работой: написанием тестов, эндпоинтов, логирования и тому подобного.
2. **Использование ИИ**. Я думал, что эта проблема меня не коснётся, но когда потребовалось добавить фронтенд-часть, оказалось, что мой backend нужно тоже переделывать. Как следствие, пришлось бы переписывать значительную часть кода. Это породило огромное нежелание заниматься переделкой. В результате я начал неосознанно "вайбкодить" (vibe-coding), что привело к печальным последствиям: нерабочим функциям, проблемам с базой данных и так далее. В итоге мне пришлось откатывать проект, но потом я все равно решил что нужно менять подход впринципе.
3. **Накопление технического долга**. Из-за чрезмерно затянувшейся первой фазы разработки я начал забывать логические связи внутри проекта. Это, в свою очередь, привело к появлению большого количества неработающего кода в одних частях после изменений в других.

Собственно, поэтому я и решил начать с «чистого листа». В этот раз мне наконец-то удастся завершить подготовительные фазы, возможно, даже в ближайшие дни. Также буду внимательнее отслеживать свою "осознанность" в процессе разработки.

## Что будет использоваться
### Основа (главное для результата и процесса)
- Docker & Docker Compose
- Nginx
- Git
- Туннель через домашний роутер
- GitLab CI & GitLab CE
- PostgreSQL
- Redis
- Health Checks API
- Telegram Bot
- Prometheus + Grafana
- Миграции базы данных в CI/CD
- Trivy
- Сервисы для которых нужна вся инфраструктура (несколько небольших для проверки работы, могут быть абсолютно любыми и на разных ЯП)
### Для дальнейшего развития (попробовать)
- Terraform
- Ansible
- Kubernetes
- Виртуальные машины (для бесплатной реализации разных серверов)
- Остальное по желанию и возможностям
## Правила (для меня как разработчика)
- Нельзя брать у ИИ готовый код (работа в терминале тоже входит) и архитектурные решения. Можно только просить объяснения на словах и объяснения должны быть тебе понятны
- Если не работал с инструментом раньше то сначала читаю документацию/получаю минимальные необходимые знания - потом внедряю
- Один коммит - одна логическая задача
- Все коммиты должны отражать суть сделанного
- Коммиты должны быть соответственно веткам в Git (main - стабильные версии (или изменения которые не влияют на работу приложения: оформление и тд), dev - основная разработка, и тд)
### По желанию
Веду **ARCHITECTURE.md** по коммитам где записываю:
1. Название коммита
2. Подробное описание: что сделал, зачем нужны изменения, почему сделал именно так
